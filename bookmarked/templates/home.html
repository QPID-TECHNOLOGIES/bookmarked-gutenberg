Project Gutenberg Bookmarked

A high-performance JSON web API for metadata & cached content of Gutenberg ebooks

API Reference

Lists of Books

Retrieve catalog data using the /books/ endpoint. Results are paginated, showing 32 books per page. By default, ordering is based on download popularity.

Example Response Structure:

{
  "count": 72480,
  "next": "http://domain/books/?page=2",
  "previous": null,
  "results": [ ... ]
}

Query Parameters

Refine your search queries using standard query parameters:

Parameter Type Description
search string Search author names and book titles. Case-insensitive. (e.g. dickens great)
languages string Comma-separated language codes. (e.g. en,fr)
author_year_start integer Filter by authors alive after this year. (e.g. 1900)
author_year_end integer Filter by authors alive before this year. (e.g. 1950)
copyright string Filter by copyright status: true, false, null.
mime_type string Filter by format types (e.g. text/html or application/epub).
sort string Sort by: popular (default), ascending, descending.

Async Content Caching

Fetch book contents dynamically using our non-blocking CDN cache wrapper to avoid Heroku timeouts.

Request content: GET /content/{gutenberg_id}/?format=epub

If cached, you'll be redirected immediately. If not, a status check endpoint will be returned: 

{
  "status": "pending",
  "gutenberg_id": 84,
  "message": "Content is being prepared. Poll /status/ for updates.",
  "status_url": "/content/84/status/?format=epub"
}
